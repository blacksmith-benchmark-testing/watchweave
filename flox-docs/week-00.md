# Onboarding Journal — Key

This file explains the tags and format used across week-01.md through
week-09.md. Doesn't change during the internship — just reference back
to this whenever needed.

## Required tags (per Project 2 instructions — pick one per entry)
- `[unclear docs]`      → documentation existed but was confusing/incomplete
- `[confusing error]`   → error message was misleading, unclear, or hard to diagnose
- `[missing feature]`   → something I expected to exist/work just isn't there
- `[expected X got Y]`  → straightforward mismatch between expectation and reality
- `[setup friction]`    → general slowdown getting something running, no single root cause

## Categories (personal use, not a Project 2 requirement — for scanning later)
- `[FLOX]` → specific to Flox/Nix itself (catalog, manifest, services, hooks)
- `[APP]`  → WatchWeave-specific (schema, env vars, app code behavior)
- `[CLI]`  → general command-line/Linux skills I'm rebuilding
- `[SYS]`  → system quirks unrelated to Flox (macOS, ports, etc.)

## Severity (optional — helps when building the week 9 memo)
🔴 blocked me for 30+ min / genuinely confusing
🟡 minor friction, resolved quickly
🟢 not really a problem, just worth recording

## Entry format

[TIMESTAMP] [TAG] [CATEGORY] [SEVERITY]
What happened:
Expected vs. got:
Resolution:
Time cost:
Notes: (optional)

Not every field needs filling every entry — "what happened" + "resolution"
is often enough. Consistency matters more than completeness on every line.

## Weekly summary habit

At the end of each week, before moving to the next file, add 2-3 lines to
themes.md summarizing recurring patterns. This makes the week 9 memo
mostly already-written by the time I get there.
